# 文档同步管理器 (Doc Sync Manager)

本技能用于自动化管理代码修改或更新后的文档生成与飞书同步,支持重构任务和新需求任务两类场景。

## 触发条件

当以下场景出现时自动触发:
- 完成页面重构工作,需要记录重构进度
- 完成新需求开发,需要记录开发总结
- 用户明确要求生成进度文档并同步到飞书

## 任务类型识别

根据用户输入自动识别任务类型:

### 重构任务 (Refactoring)
识别关键词:
- "重构"、"重构完成"、"refactor"
- "优化"、"代码优化"、"性能优化"
- "迁移"、"升级"、"改造"
- "重写"、"改写"

### 新需求任务 (New Feature)
识别关键词:
- "新需求"、"需求完成"、"feature"
- "新增功能"、"开发完成"
- "实现"、"添加功能"

## 文档命名规范

### 文件名格式
```
{日期}_{任务类型}_{模块/页面名称}.md
```

示例:
- `2025-05-14_重构_用户中心页面.md`
- `2025-05-14_新需求_订单导出功能.md`

### 飞书文档标题格式
```
[{任务类型}] {模块/页面名称} - {日期}
```

示例:
- `[重构] 用户中心页面 - 2025-05-14`
- `[新需求] 订单导出功能 - 2025-05-14`

## 工作流程

### 1. 信息收集

向用户确认以下信息:
- **任务类型**: 重构 / 新需求
- **模块/页面名称**: 具体修改的模块或页面
- **修改内容概述**: 简要描述做了什么修改
- **涉及文件列表**: 修改了哪些文件
- **关键改动点**: 重要的代码变更或逻辑调整
- **测试结果**: 是否已完成测试
- **备注**: 其他需要记录的信息

### 2. 文档生成

根据任务类型选择对应模板生成文档:

#### 重构文档模板

```markdown

- **任务类型**: 重构
- **模块/页面**: {模块名称}
- **完成日期**: {日期}
- **执行人**: {执行人}

{重构原因和背景说明}

### 涉及文件
{文件列表}

### 关键改动
1. {改动点1}
2. {改动点2}
...

## 代码变更摘要
```
{关键代码片段或变更说明}
```

## 测试情况
- [ ] 单元测试
- [ ] 集成测试
- [ ] 回归测试
- [ ] 性能测试

**测试结果**: {测试结论}

## 风险评估
- **影响范围**: {影响的功能模块}
- **回滚方案**: {如有问题如何回滚}
- **注意事项**: {上线后需要关注的事项}

## 后续优化建议
{可选，列出可以进一步优化的方向}
```

#### 新需求文档模板

```markdown
# [{需求名称}] 开发总结文档

- **任务类型**: 新需求
- **需求名称**: {需求名称}
- **完成日期**: {日期}

1. {核心逻辑点1}
2. {核心逻辑点2}
...

## 代码实现
```
{关键代码实现}
```

## 功能验证

### 测试用例
- [ ] {测试用例1}
- [ ] {测试用例2}
...

### 验收标准
- [ ] {验收标准1}
- [ ] {验收标准2}
...

**验证结果**: {验证结论}

{功能使用方法和注意事项}

- **依赖项**: {新增依赖}

```

### 3. 本地保存

- 将生成的文档保存到 `/workspace/documents/` 目录
- 文件名遵循命名规范

### 4. 飞书同步

使用飞书技能将文档同步到飞书:

1. **创建飞书文档**
   - 使用 `lark-doc` 技能创建文档
   - 文档标题遵循命名规范
   - 内容使用生成的 Markdown

2. **设置文档权限** (可选)
   - 根据用户要求设置分享权限
   - 添加协作者

3. **返回分享链接**
   - 获取飞书文档分享链接
   - 返回给用户确认

## 依赖工具

本技能依赖以下工具:
- `lark-doc` 技能: 用于创建和编辑飞书文档
- `lark-cli`: 飞书命令行工具

## 使用示例

**示例1 - 重构任务:**
```
用户: "用户中心页面重构完成了，帮我生成进度文档同步到飞书"
→ 识别为重构任务
→ 收集信息：模块名称、修改内容、涉及文件等
→ 生成重构文档
→ 同步到飞书
→ 返回分享链接
```

**示例2 - 新需求任务:**
```
用户: "订单导出功能开发完了，记录一下并上传到飞书"
→ 识别为新需求任务
→ 收集信息：需求名称、实现方案、测试情况等
→ 生成新需求文档
→ 同步到飞书
→ 返回分享链接
```

## 注意事项

1. **信息完整性**: 确保收集到足够的信息来生成有意义的文档
2. **敏感信息**: 提醒用户不要在文档中包含密码、密钥等敏感信息
3. **文档权限**: 确认飞书文档的分享范围和权限设置
4. **文件备份**: 本地保存的文档作为备份，方便后续查阅
